# Key Characteristics of an Effective Work Breakdown Structure (WBS)

A good WBS is hierarchical, representing all deliverables for physical products, services, or processes. It may include project methodology deliverables, often in a separate methodology WBS. Each level adds granularity, with child layers clarifying parent deliverables. The WBS must be complete (100% rule) and agreed upon by all stakeholders. It serves as the foundation for defining activities, schedules, budgets, and scope‑creep control. The WBS is applicable to any project type because every project produces deliverables.
