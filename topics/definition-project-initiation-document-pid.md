# Project Initiation Document (PID): Definition and Core Contents

A Project Initiation Document (PID) captures the essential information required to start a project on the right footing. It records the project’s context, objectives, scope, governance, success metrics, stakeholder roles, and any constraints. The PID serves both as an internal guide for the project team and as a reference for external stakeholders. It is typically created after the project has been signed off by the client, resources are secured, and initial planning is completed. The PID can be a living document that evolves as the project progresses, helping maintain alignment and providing a single source of truth for decisions.
