# Work Breakdown Structure (WBS) and Resource Allocation Guide

Break the project into smaller work packages, map each package to dates and responsible resources. A high‑level WBS example shows phases (e.g., discovery, design, development, testing) and associated team members. Involve the team during planning to surface dependencies and improve accountability.

## Can Answer

- What is a Work Breakdown Structure (WBS)?
- How do I allocate resources using a WBS?
- How can a WBS improve project planning?

## Examples

- Phase 1 – Discovery (2 weeks) – Lead: Business Analyst, Support: UX Designer.
- Phase 2 – Development (8 weeks) – Lead: Front‑end Engineer, QA: Tester.

## Warnings

- A WBS that is too high‑level can hide critical task dependencies.
