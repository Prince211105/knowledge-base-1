# Document Risks, Assumptions, Issues, and Dependencies in a PID

Document known risks (e.g., tight timeline, budget constraints, technical unknowns), assumptions (e.g., client will supply assets on time), issues (current problems), and dependencies (external deliverables). Provide mitigation or contingency plans for each risk.

## Can Answer

- What risks should be listed in a PID?
- How do I document assumptions in a PID?
- How can I record project dependencies in a PID?

## Examples

- Risk: Vendor delay – Mitigation: Identify backup vendor.
- Assumption: All APIs will be stable during the integration phase.

## Warnings

- Failing to list realistic risks can leave the team unprepared for setbacks.
