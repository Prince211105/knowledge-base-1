# Step‑by‑Step WBS Decomposition Process

The decomposition begins with the top‑level deliverable (Level 1) that groups major product, service, or process outcomes. Stakeholders drill down through parent deliverables, adding child components that clarify the parent. Completeness is checked by asking whether any solution concept cannot be associated with an existing deliverable; if so, a new parent deliverable is added. Conversely, items not part of the intended solution are removed. The process continues until the lowest level can only be expressed as verb phrases, signalling the transition to activity definition.
