# What the WBS contains and what it excludes

## Overview

Lists the information that belongs in a WBS and the information that does not.

## Detailed Summary

The WBS contains deliverables and deliverable components representing the total project scope. It may also include intermediary project methodology outcomes required to manage the project effort. It does not contain project activities, tasks, milestones to be achieved, timing dependencies, durations, resource assignments, cost estimates, or network logic. These other planning elements belong to the Project Schedule, Network Diagrams, and Project Budget.

## Content

The WBS includes all deliverables for the final product, service, or process being produced by the project. It can also include project methodology deliverables needed to manage and complete project execution, although some teams document those in a separate project methodology WBS. The WBS excludes activities, tasks, milestones, timing dependencies, durations, resource assignments, and costs. Those items are developed later and belong in the project schedule, network diagrams, and budget. The WBS is agnostic to timing, effort, and costs; it is concerned only with what must be produced.

- Whether a specific item belongs in the WBS
- How to separate WBS content from schedule content
- How to separate WBS content from budget content

## Best Practices

- Review each proposed WBS element and test whether it is a deliverable rather than a work action.
- Keep timing, sequencing, and costing data in their appropriate planning documents.

## Examples

- Deliverable: Frame Set belongs in the WBS. Activity: Paint Frame belongs in the schedule.

## Warnings

- Including activities or costs in the WBS blurs the distinction between scope definition and execution planning.

## Limitations

- A WBS alone cannot manage sequencing, staffing, or cost control without related planning artifacts.

## Troubleshooting

- If the WBS starts to resemble a task list, replace verb phrases with deliverable nouns and move actions to the project schedule.

## Decision Rules

- Include nouns that represent outputs.
- Exclude verbs, dates, durations, dependencies, and costs.
