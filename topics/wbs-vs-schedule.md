# Comparing Work Breakdown Structure (WBS) and Project Schedule

A Work Breakdown Structure (WBS) is a hierarchical, deliverable‑oriented decomposition of the total scope of work. It lists *what* must be produced using noun‑based statements and does not contain timing, effort, or cost data. A Project Schedule, in contrast, is an activity‑oriented plan that details *how* the deliverables will be produced. It includes verb‑based task descriptions, durations, resource assignments, dependencies, milestones, and cost information. The WBS serves as the blueprint for the schedule: once the lowest‑level WBS elements can no longer be expressed as nouns, they are broken down into activities that become the schedule entries.
