# Guidelines for creating an effective WBS

## Overview

Lists the document's recommended guidelines for producing an effective WBS.

## Detailed Summary

The document recommends several guidelines for an effective WBS: use nouns rather than verbs to focus on deliverables, use a hierarchy to illustrate deliverable relationships, build the WBS collaboratively with people who will produce the deliverables, ensure each WBS leg contains unique deliverables rather than repeating the same deliverable in multiple legs, never assume stakeholders will know a deliverable is part of the project if it is not shown, and follow the 100% rule so each level fully represents its parent deliverable.

## Content

Effective WBS creation follows a set of practical rules. Keep the focus on deliverables by using noun phrases. Structure the WBS hierarchically so the relationships between deliverables are visible. Build it collaboratively with the people who will actually produce the deliverables. Make each branch unique to avoid redundant scope representation. Explicitly include all needed deliverables rather than relying on assumptions. And apply the 100% rule at every level so child elements fully account for the parent deliverable.

## Best Practices

- Use noun-based labels.
- Use hierarchical structure.
- Build collaboratively.
- Keep each leg unique.
- Do not rely on unstated assumptions.
- Apply the 100% rule.

## Examples

- Instead of repeating the same deliverable under multiple branches, place it once in the most logical unique leg.

## Warnings

- Repeated deliverables across branches can create ambiguity and double counting.
- Omitted deliverables may be forgotten in planning if they are not shown.

## Limitations

- Following guidelines does not eliminate the need for stakeholder review and baseline approval.

## Troubleshooting

- If the WBS has duplicates, consolidate them into a unique branch.
- If stakeholders say an omitted item is 'understood,' add it explicitly instead of assuming.

## Decision Rules

- Only include explicitly stated deliverables.
- Ensure each branch uniquely represents its part of scope.
- Verify every child set fully represents its parent.
