# How a Work Breakdown Structure Supports Scope, Schedule, and Budget Management

The Work Breakdown Structure (WBS) creates the scope baseline by listing every deliverable and work package. This detailed breakdown is the primary input for defining activities, which in turn form the project schedule (activities, durations, dependencies). Because each work package is cost‑assigned, the WBS also serves as the foundation for cost estimating and the project budget. When a change request alters scope, the WBS is updated, causing automatic adjustments to activities, schedule, and budget, ensuring alignment across all planning dimensions.
