# 100% rule in WBS decomposition

## Overview

Defines the 100% rule and explains how it is used when drilling down through WBS levels.

## Detailed Summary

The 100% rule states that every level of decomposition in the WBS must contain all of the deliverable elements that represent 100% of its parent deliverable. It should be applied continuously when decomposing the WBS. Teams should keep asking whether the new level of detail represents everything that makes up the parent deliverable above it.

## Content

The 100% rule is a completeness rule for WBS decomposition. Each set of child deliverables must fully account for the parent deliverable, with no missing scope. As the WBS is drilled down, stakeholders should verify that the lower level captures the full content of the higher level. This rule supports completeness, reduces omissions, and ensures the hierarchy remains a faithful decomposition of project scope.

## Best Practices

- Test every parent-child breakdown against the 100% rule.
- Use the rule repeatedly at each level rather than only at the end.

## Examples

- When decomposing Training Materials, the child deliverables should together represent all of Training Materials, not just selected parts.

## Warnings

- Ignoring the 100% rule can leave missing scope hidden within WBS branches.

## Limitations

- The rule ensures completeness of decomposition but does not dictate the best organizational structure.

## Troubleshooting

- If a parent seems only partially explained by its children, add the missing child deliverables or reconsider the branch structure.

## Decision Rules

- A decomposition level is acceptable only if its child elements together represent 100% of the parent deliverable.
