# Work Breakdown Structure definition and primary purpose

## Overview

Defines the Work Breakdown Structure and explains its primary purpose as a deliverables-based scope artifact.

## Detailed Summary

A Work Breakdown Structure (WBS) is a hierarchical decomposition of the total scope of work to be carried out by the project team to accomplish project objectives and create required deliverables. Its purpose is to represent all deliverables to be produced by the project, not to list activities, costs, durations, dependencies, or resource assignments. The WBS answers the question, "What are we going to produce as a result of the project?" and serves as a higher-level artifact that supports creation of schedules, budgets, and other planning outputs.

## Content

The WBS is a higher-level project artifact focused on deliverables. It represents all of the deliverables the project will produce and provides the basis for later defining activities, estimating costs, sequencing work, and building schedules and budgets. It is not itself a scheduling or budgeting deliverable. The WBS exists to establish agreement on project scope by documenting what the project will produce. It answers the core question: "What are we going to produce as a result of the project?" The answer is the set of WBS deliverables.

## Best Practices

- Use the WBS to establish shared understanding of project deliverables.
- Treat the WBS as the main artifact for communicating project scope.

## Examples

- A WBS for a mountain bike project would list deliverables such as bike components rather than tasks to build them.

## Warnings

- Do not confuse the WBS with a project schedule or budget.

## Limitations

- The WBS does not describe how work is sequenced or timed.

## Troubleshooting

- If stakeholders describe activities, costs, or dates instead of outputs, refocus the discussion on deliverables.

## Decision Rules

- If an item describes what will be produced, it may belong in the WBS.
- If an item describes how, when, who, or how much, it belongs in another planning artifact rather than the WBS.
