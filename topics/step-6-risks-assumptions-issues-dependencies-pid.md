# Step 6: Document Risks, Assumptions, Issues, and Dependencies

Create a concise table or list that captures:
- **Risks**: Potential events that could affect success, with probability, impact, and mitigation strategy.
- **Assumptions**: Planning premises taken as true (e.g., client provides assets within five days).
- **Issues**: Existing problems requiring resolution.
- **Dependencies**: Tasks that rely on external inputs or other project phases.
This overview supports clear stakeholder communication and early action.


## Can Answer

What information belongs in the Risks, Assumptions, Issues, and Dependencies section of a Project Initiation Document?

## Examples

- Risk: Vendor API may change during development – Mitigation: Allocate a 10 % buffer for integration testing.

## Warnings

- Including risks without associated mitigation actions reduces the usefulness of the register.
