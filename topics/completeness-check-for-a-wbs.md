# Completeness check for a WBS

## Overview

Provides a method for checking whether the WBS fully covers the intended project solution.

## Detailed Summary

A practical way to assess WBS completeness is to ask stakeholders whether there is any project solution concept that cannot be associated with one of the deliverables currently shown in the WBS. If something is missing and cannot be categorized under an existing deliverable, it likely represents a new parent-level deliverable. If something appears in the WBS but is not part of the intended solution, it must be removed as out of scope.

## Content

During decomposition, stakeholders should test the WBS for completeness by asking whether every concept required by the intended project solution can be placed somewhere in the current WBS. If an element cannot be associated with any existing deliverable, the WBS is incomplete and a new deliverable branch may be needed. If an included element is not actually part of the intended solution, it should be removed. This review helps achieve stakeholder alignment on everything the project must provide before design and execution begin.

## Can Answer

- How to validate WBS completeness

## Best Practices

- Ask stakeholders to map every solution concept to the WBS.
- Remove anything that does not belong to the intended project solution.

## Examples

- If a bike component cannot be categorized under any existing branch, a new top-level or parent deliverable may be needed.

## Warnings

- Assuming stakeholders will infer missing deliverables can create scope gaps.

## Limitations

- Completeness depends on stakeholder knowledge and the current understanding of the solution.

## Troubleshooting

- If stakeholders identify uncategorized solution elements, add or revise WBS branches.
- If an element seems unnecessary, confirm whether it is truly part of the approved scope before keeping it.

## Decision Rules

- If a required concept cannot fit under an existing branch, create or revise a parent deliverable.
- If an element is not part of the intended solution, remove it from the WBS.
