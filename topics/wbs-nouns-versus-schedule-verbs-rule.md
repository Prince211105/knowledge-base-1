# WBS nouns versus schedule verbs rule

## Overview

Provides the simple language rule for keeping a WBS deliverables-based.

## Detailed Summary

To ensure the WBS remains deliverables-based rather than activities-based, every element in the WBS should be represented as a noun phrase rather than a verb phrase. The rule is: WBS equals nouns, Schedule Activities equals verbs. When decomposition shifts from nouns describing a deliverable to verb phrases describing how it will be produced, the WBS has reached its lowest useful level of decomposition.

## Content

The simplest test for whether an item belongs in the WBS is grammatical. If the item is a noun phrase, it likely represents a deliverable and may belong in the WBS. If it is a verb phrase, it likely describes an activity and belongs in the schedule. This rule helps stakeholders maintain a clear distinction between scope definition and execution planning and signals when to transition from WBS decomposition to activity definition.

## Best Practices

- Review WBS labels and rewrite verb phrases as noun phrases where possible.
- Use the nouns-versus-verbs rule during workshops to prevent activity drift.

## Examples

- Frame is a WBS noun. Paint Frame is a schedule verb phrase.

## Warnings

- Verb-based WBS elements indicate the team may be building a schedule instead of a WBS.

## Limitations

- A noun phrase alone does not guarantee the item is in scope or sufficiently clear.

## Troubleshooting

- If a WBS branch contains phrases like design, build, review, or install as actions, move them to activity planning and replace them with deliverable names.

## Decision Rules

- If the element is a noun phrase representing an output, place it in the WBS.
- If the element is a verb phrase representing an action, place it in the schedule.
