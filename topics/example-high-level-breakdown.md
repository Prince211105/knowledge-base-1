# High‑Level Project Breakdown Structure (PBS) Example

High‑level PBS Example:
- **Phase 1 – Discovery (2 weeks)**
  - Activity: Stakeholder interviews – 1 Business Analyst (16 h)
  - Activity: Requirements gathering – 1 BA (24 h)
  - Activity: Initial wireframes – 1 UX Designer (20 h)
- **Phase 2 – Development (8 weeks)**
  - Activity: Front‑end coding – 2 Developers (320 h)
  - Activity: Back‑end API – 2 Developers (320 h)
  - Activity: QA testing – 1 QA Engineer (80 h)
- **Phase 3 – Launch (2 weeks)**
  - Activity: User acceptance testing – 1 QA + Client PM (40 h)
  - Activity: Deployment – 1 DevOps Engineer (16 h)
This structure aligns with the project timeline and can be expanded with detailed tasks.
